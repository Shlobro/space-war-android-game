# Game Test Package Guide

## Purpose

`app/src/test/java/com/example/cw/game` contains JVM tests for deterministic gameplay helpers in `com.example.cw.game`.

## Files

- `CampaignPersistenceTest.kt`: Verifies round-trip serialization for completed levels and per-level star records, defensive parsing of malformed persisted data, current spent-star persistence, the persisted campaign upgrade levels for mission income, player-node refill rate, and player fleet speed, and schema-version gating so unsupported save formats fall back to a safe empty campaign state while older save formats still load without inventing extra spendable stars. The file also pins the explicit migration rules that zero `Cash Flow` for pre-v2 saves and ignore refill/speed upgrade values for pre-v3 saves.
- `GameAppBackNavigationTest.kt`: Verifies the top-level back-navigation resolver keeps Android system back aligned with the visible app flow by leaving `HOME` unhandled for the activity, returning `LEVELS` and `UPGRADES` to `HOME`, pausing a running mission on the first back press, and exiting paused, completed, or inconsistent in-game states to the level list.
- `LevelCardStyleTest.kt`: Verifies the menu-only level-card style helpers return the intended high-contrast palette for completed and unlocked missions, checks that the locked-card text still clears explicit contrast thresholds against the locked surface, pins the palette-owned status letter spacing, covers the null-safe generic locked label, and rejects impossible "completed but locked" state combinations.
- `GameCanvasTest.kt`: Verifies badge and overlay rendering helpers that do not need a device. The file covers overlapping badge geometry on normal and small rendered node sizes, density-safe badge text sizing bounds for representative small and medium nodes, bottom-edge badge visibility clamps, the small upward upgrade-indicator geometry and top-edge clamp for affordable player nodes, the affordability/max-level gating helper for that indicator, and upgrade-button overlay offsets near viewport edges and safe insets.
- `GameTapLaunchMessageTest.kt`: Verifies tap-triggered attacks and friendly reinforcements are silent on success, covers both single-source and multi-source selections, checks mixed-success plus all-sources-fail launch batches, and pins that a valid launch clears any transient tap hint instead of leaving stale onboarding text visible.
- `GameLogicTest.kt`: Verifies fleet launch math, silent successful player launches, player base upgrades, the round-number upgrade cost schedule and insufficiency messaging, AI spending against that same upgrade schedule, obstacle-aware route generation, reduced passive-funds accrual with explicit arithmetic notes, campaign fleet-speed multipliers on player launches, campaign refill-rate multipliers on player-owned node production only, campaign-upgrade purchase eligibility with both star and max-level guards, capture-level downgrade behavior after successful assaults including over-cap survivor cases, friendly-arrival over-cap reinforcement handling plus later decay toward cap on follow-up ticks, tap-selection behavior that keeps HUD messages free of selection text while also clearing selection on empty-space taps or stale non-player selections, short-lived onboarding hints for invalid enemy-target taps, preservation of those hints until their configured expiry even when the player immediately taps a valid base, selected-base upgrade visibility at `maxLevel`, compact HUD summary formatting for the floating player-funds readout and live mission timer, formatter clamping for negative elapsed time, total-minute formatting beyond one hour, pause toggling for the HUD action, pause-safe match stepping, star-award thresholds, campaign best-star accumulation, spendable-star accounting, the post-victory best-star delta flow plus transition-safety for already-finished wins, selection cleanup after ownership changes during simulation, and the five-second enemy AI think cadence plus capped-out max-level AI pressure attacks only when the odds are reasonable.
- `GameWorldRouteTest.kt`: Verifies `GameWorld` route building keeps every returned segment outside the inflated obstacle buffer for both single-obstacle and multi-obstacle layouts, checks obstacle avoidance near the named world clamp margins, confirms the helper still returns the direct endpoint when no obstacle blocks the straight segment, and documents the intentional endpoint-only fallback when no graph path can be built inside the constrained world bounds.
- `UpgradeAnimationTest.kt`: Verifies the UI helper that detects when a selected player base has just leveled up, ensures selection swaps do not trigger the animation, keeps the distinction between a selected player base and an actually upgradable one explicit so max-level upgrades can still animate on their final tier, and covers the extracted animation runner so upgrade-ring overlay state is cleared both after a normal completion and after coroutine cancellation.

## Subfolders

- `levels/`: Tests for JSON parsing, authored level validation, summary sorting, unlock rules, and authored-to-runtime mapping. See `levels/levels_test_developer_guide.md`.

## Working In This Folder

- Add tests here when the production code can run without Android framework services.
- Prefer small state fixtures and direct calls into extracted helpers so failures point to a single gameplay rule.
- Keep base fixtures internally consistent with authored rules, including `cap = capLevel * 10`.
